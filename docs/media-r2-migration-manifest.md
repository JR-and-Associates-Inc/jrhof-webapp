# Media R2 Migration Manifest

Working record for branch `chore/media-r2-optimization`. This branch **prepares**
optimized media and its R2 delivery plan; it does **not** upload to R2, change any
live `portrait_url`, or deploy. The live inductee → R2 cutover remains the gated
step described in [INDUCTEE_MEDIA_R2_MIGRATION.md](INDUCTEE_MEDIA_R2_MIGRATION.md).

Related, unchanged infrastructure: golf 2024/2025/2026 and the 2026 banquet
gallery are already on R2 via `media.jrhof.org` (see
[R2_MEDIA_MIGRATION.md](R2_MEDIA_MIGRATION.md)); those manifests live under
`manifests/r2/`. This branch does not touch galleries or the protected
`feature/banquet-registration-checkout` branch.

## Inductee portrait R2 object plan

Source → optimized → R2, generated by `scripts/optimize-inductee-portraits.mjs`.
Full per-record entries (source SHA-256, dimensions, bytes, output SHA-256) are in
`manifests/r2/inductee-portraits-v1.json`.

- Bucket: `jrhof-media-public` (public, documented; not a secret).
- Media origin: `https://media.jrhof.org`.
- Content-Type: `image/webp`.
- Cache-Control: `public, max-age=31536000, immutable`.
- Object keys are **versioned and immutable** (`…/v1/…`). A corrected portrait
  publishes under a new version key (`v2`); bytes behind a `v1` URL are never replaced.

| Variant | Consumers | Width cap | Quality | Object key template |
|---|---|---:|---:|---|
| `profile.webp` | biography portrait, Open Graph, `Person.image` | 960 px | 88 | `inductees/portraits/v1/<slug>/profile.webp` |
| `card.webp` | archive / homepage / banquet cards | 400 px | 82 | `inductees/portraits/v1/<slug>/card.webp` |
| placeholder | 33 pending records | 400 px | 82 | `inductees/placeholders/v1/missing-inductee.webp` |

- 117 verified records × 2 variants + 1 placeholder = **235 objects**, ~7.7 MB total.
- Portraits are resized by width with `withoutEnlargement` (no upscaling) and are
  **not** force-cropped — faces are preserved; the UI applies `object-fit: cover`.
- EXIF/GPS/XMP/ICC metadata is stripped (sharp default WebP encoder); `.rotate()`
  bakes orientation first. Verified: sample outputs report no EXIF/XMP/ICC.
- 33 pending / board-review-blocked records mint **no** object key (`pendingRecords`
  in the manifest). Six unreferenced quarantined JPEGs are excluded pending identity review.

Example resolved URL:
`https://media.jrhof.org/inductees/portraits/v1/terry_angell/profile.webp`

### Determinism note

WebP output bytes depend on the local libvips build shipped with `sharp`. The
manifest records output SHA-256 for pre-upload verification (`verify-local`); a
hash difference on a different `sharp` build is expected — regenerate to refresh.
Source SHA-256 and object keys are stable identity.

## Repo cleanup — untracked originals (task 11)

`git rm --cached -r content/Photos` untracked **156** inductee source originals
(11.9 MB): 117 verified portrait sources + 39 pending/"missing" markers.

- `/content/` was already in `.gitignore`; the files were tracked from before that
  rule and now stop being recommitted.
- Binaries remain **on local disk** and in **Git history** — fully reversible
  (`git checkout <prev> -- content/Photos` to restore tracking).
- Provenance (path, bytes, SHA-256, referencing records) is preserved in
  `manifests/audits/untracked-content-originals.json`.
- Canonical home for these originals is the controlled JR & Associates Google Drive
  / R2 archive, per [MEDIA_STRATEGY.md](MEDIA_STRATEGY.md) (Git is not the archive).
- The site build does **not** read `content/` at runtime (biography text is inlined
  in `src/data/inductees.json`; portraits are served from `public/images/inductees/`),
  so the build is unaffected — verified by `npm run build`.

Published web portraits under `public/images/inductees/*.jpg` remain **tracked**:
they are the live-served assets and must stay until the R2 cutover.

   `Person.image`) to the media origin **together**, preserving placeholder behavior.

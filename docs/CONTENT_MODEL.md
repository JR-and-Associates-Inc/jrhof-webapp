# Inductee Content Model

## Location

The candidate dataset is `src/data/inductees.json`. It is generated by `scripts/generate_inductee_candidates.py` from:

- `content/All Inductees.csv`
- `_migration/source-reconciliation/updated-inductee-source-status.csv`
- `_migration/reconciliation/inductee-reconciliation.json`
- readable files under `content/Bios/`
- verified-candidate files under `content/Photos/`

## Record fields

Each of the 150 records contains:

- `stable_id`
- `display_name`, `sort_name`, and `aliases`
- `induction_year` or `induction_era`
- `canonical_slug`
- `current_wordpress_url`
- `proposed_canonical_url`
- `legacy_repo_url` and `redirect_sources`
- `bio_status`, `bio_source`, and extracted `biography` paragraphs
- `portrait_status`, `portrait_source`, `portrait_output_filename`, and `portrait_url`
- `live_link_status`
- `migration_lane`
- `publication_status`
- `board_review_required`
- `reviewer_notes`
- `source_provenance`

## Publication behavior

- `bio_status=available`: render extracted original-source paragraphs.
- `bio_status=pending_review`: render “Biography pending review.”
- `portrait_status=verified_candidate`: render the copied person-specific source photo.
- `portrait_status=pending_review`: render `/images/inductees/portrait-pending.svg`.
- `board_review_required=true`: show a review notice without exposing internal notes as asserted facts.

## Special cases

- Robert Schnabel is sourced only from `content/Bios/Robert_Schnabel.docx`; the WordPress/live biography is prohibited.
- Terry/Ray Garvey remains identity blocked and uses the shared placeholder.
- Sam Corentino/Corsentino retains aliases and review status; the identity-sensitive source portrait is not promoted yet.

## Regeneration

Run `npm run content:generate`, then `python3 scripts/generate_redirects.py`. Review the diff before committing because source approvals can change publication status and routes.
